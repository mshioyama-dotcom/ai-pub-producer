## 1. サービスコンセプト

**「出版して終わり」ではなく「出版を起点に稼ぎ続ける」**

使えば使うほど自分専用になり、やめるほど損をする循環型サービス。

## 2. ターゲット

出版を集客・権威づけに使いたい事業者・副業者。
複数冊のシリーズ展開を目指す人。

## 3. サービスの核心思想

**「AIは材料を出す、判断は人が行う」**

コミュニティでの人の気づき（売れるパターンの発見）がツールに還元される。
使い続けるほどツールの精度が上がり、サービス全体の価値が高まる。

```
【完全な循環設計】

研究会でベストセラー分析・議論
　↓
売れるパターンを言語化
　↓
ツール（Difyプロンプト）をアップデート
　↓
受講生のアウトプット品質が上がる
　↓
また新しい分析へ
```

## 8. 売れる本研究会（月次Zoom）

### 概要

まさしおさんの商業出版・Kindle15冊以上の経験を活かした「売れるパターン」の研究会。
ツールでは補えない「なぜ売れるか」の質的判断を人の議論で補う。

### 毎月の流れ（60分）

```
事前（Slack）
　まさしおさんが今月の分析対象本を告知
　参加者が各自で読んでくる（orサマリーを共有）
　↓
当日（Zoom・60分）
　①今月のベストセラー分析（20分）
　　・なぜ売れたか・タイトル・表紙・説明文・レビュー分析
　　・市場トレンドとの関係
　②参加者の企画フィードバック（30分）
　　・「売れるか」の視点でまさしおさんが一言コメント
　③来月の予告（10分）
　↓
事後（Slack）
　議論のまとめをまさしおさんが投稿
　参加者が自分の企画に応用した気づきをシェア
```

### ツールへのフィードバックループ

研究会での気づきを定期的にDifyプロンプトに反映する。

```
研究会で「このパターンが売れる」と判明
　↓
STEP1・2・5などの該当プロンプトを更新
　↓
受講生が更新されたツールで精度向上を体感
```

## 9. 相互応援の仕組み

強制にせず「やりたくなる仕組み」として設計する。

### ①相互レビューの文化
- 出版したらSlackで報告 → メンバーがAmazonレビューを書く
- 自分が出版したときに応援してもらえる返報性が機能する
- まさしおさんが率先してレビューを書く文化を作る

### ②出版報告→応援投稿の連鎖
- X投稿でメンバーの本を紹介し合う
- STEP12に「メンバーの本を紹介するX投稿文生成」を追加
- 応援の敷居を下げてツールが後押しする

### ③実績の可視化（Slack）
- 今月出版した人
- レビュー数が増えた人
- ランキング上昇した人

可視化することで自然に応援したくなる空気を作る。

## 10. STEP0：著者プロファイル設定（新規追加・最重要）

### 概要

過去のアウトプットからAIが著者プロファイルを自動生成、またはインタビュー形式で生成。
生成されたプロファイルはSTEP1〜14の各プロンプトに自動注入される。

### 画面フロー

```
「過去のアウトプットはありますか？」
　↓
あり → パターンA（テキスト貼り付け）
なし → パターンB（チャットインタビュー）
　↓
共通フォーマットでプロファイル生成
　↓
「プロファイルを保存」→ STEP1へ
```

### パターンA：アウトプットあり

**入力：** 以下のいずれか1つ以上を貼り付け

| 入力欄 | 内容 |
|--------|------|
| 過去の著作テキスト | Kindle・商業出版の原稿テキスト |
| X投稿まとめ | 過去のX投稿をまとめたテキスト |
| ブログ・note記事 | 記事テキストをそのまま貼り付け |
| メルマガ | バックナンバーのテキスト |

**処理：** AIが6項目を自動抽出してプロファイルを生成

### パターンB：アウトプットなし

STEP1と同じチャット形式。AIが以下6項目を引き出すインタビューを実施。

| 質問 | 抽出する項目 |
|------|------------|
| Q1. 専門領域・強みを教えてください | 専門領域・強み |
| Q2. どんな人に読んでほしいですか | ターゲット読者像 |
| Q3. 仕事や副業で大切にしている考え方は？ | 思想・価値観 |
| Q4. よく使う言葉や表現はありますか | 文体・フレーズ |
| Q5. 経歴・実績を簡単に教えてください | 実績・経歴サマリー |
| Q6. どんな著者ブランドを目指していますか | ブランドコンセプト・シリーズ方針 |

### 出力（共通フォーマット）

```
【著者プロファイル】

■ 専門領域・強み
（テキスト）

■ ターゲット読者像
（テキスト）

■ 思想・価値観
（テキスト）

■ 文体・フレーズ
（テキスト）

■ 実績・経歴サマリー
（テキスト）

■ ブランドコンセプト・シリーズ方針
（テキスト）
```

### 各STEPへの注入方式

| STEP | 注入する項目 |
|------|------------|
| STEP1 テーマ発見 | 専門領域・強み／ブランドコンセプト・シリーズ方針 |
| STEP2 市場勝率診断 | 注入なし（客観データ処理のため） |
| STEP3 読者・価値設計 | ターゲット読者像／思想・価値観 |
| STEP4 エピソードインタビュー | 実績・経歴サマリー／専門領域・強み |
| STEP5 タイトル作成 | ブランドコンセプト・シリーズ方針／文体・フレーズ |
| STEP6〜8 目次・構成・プロット | 思想・価値観 |
| STEP9 本文作成 | 文体・フレーズ／思想・価値観 |
| STEP10〜11 説明文・A+コンテンツ | ブランドコンセプト・シリーズ方針／実績・経歴サマリー |
| STEP12 X投稿・プレゼント企画 | 文体・フレーズ／ターゲット読者像 |
| STEP13 出版状況分析 | 注入なし（客観データ処理のため） |
| STEP14 次回作構想 | 専門領域・強み／ブランドコンセプト・シリーズ方針／ターゲット読者像 |

**Difyプロンプトへの追加方法：**
```
各STEPのプロンプト冒頭に以下を追加：

以下は著者プロファイルです。このプロファイルを踏まえて出力してください。
{author_profile}  ← 該当項目のみ注入
```

**ポータル側の対応：**
- 各STEPの入力欄に「著者プロファイルを自動転記」ボタンを追加
- STEP9の「著者の過去の執筆データ」欄はSTEP0プロファイルで代替（欄は残す）

### 更新タイミング

STEP14完了後、「プロファイルを更新しますか？」の確認を経てSTEP0に戻る。
今回の本のアウトプットを追加入力することでプロファイルが進化する。
